h337

“h337”是heatmap.js注册的全局对象的名称。您可以使用它来创建热图实例

h337.create(configObject)

返回heatmapInstance。

使用h337.create创建热图实例。可以使用configObject自定义Heatmap。
configObject参数是必需的。

可能的配置属性：

  • container(DOMNode) required
    应该附加热图画布的DOM节点(热图将适应节点的大小)
  • backgroundColor(string) optional
    十六进制，颜色名称或rgb(a)形式的背景颜色字符串
  • gradient(object) optional
    表示渐变的对象(语法：数字字符串[0,1]：颜色字符串)，查看示例
  • radius(number)可选
    每个数据点将具有的半径(如果未在数据点本身上指定)
    不透明度(number)[0,1] 可选默认= .6
    整个热图的全局不透明度。如果设置，这将覆盖maxOpacity和minOpacity！
  • maxOpacity(number)[0,1] optional
    热图中最高值的最大不透明度。(如果设置了不透明度，将被覆盖)
  • minOpacity(number)[0,1] 可选
    热图中最低值的最小不透明度(如果设置了不透明度，将被覆盖)
  • onExtremaChange函数回调
    传递回调以接收极值更改更新。对DOM传说很有用。
    模糊(number)[0,1] 可选默认= 0.85
    将应用于所有数据点的模糊因子。模糊因子越高，渐变越平滑
  • xField(string) optional default =“x”
    数据点中x坐标的属性名称
  • yField(string) optional default =“y”
    数据点中y坐标的属性名称
  • valueField(string) optional default =“value”
    数据点中y坐标的属性名称
    示例配置
    使用标准渐变进行简单配置

自定义渐变配置

heatmapInstance

h337.create返回Heatmap实例。热图实例具有自己的内部数据存储区和渲染器，您可以在其中操作数据。结果，热图得到更新(部分或完全，取决于是否有必要)。

heatmapInstance.addData(对象|阵列)

返回heatmapInstance

仅将此功能用于动态添加数据点，而不是用于数据初始化！heatmapInstance.addData将单个或多个数据点添加到热图的数据存储区。

heatmapInstance.setData(对象)

返回heatmapInstance

使用数据集初始化热图实例。需要“min”，“max”和“data”属性。
setData从热图实例中删除所有先前存在的点，并重新初始化数据存储区。

heatmapInstance.setDataMax(数字)

返回heatmapInstance

更改数据集的上限并触发完全重新呈现。

heatmapInstance.setDataMin(数字)

返回heatmapInstance

更改数据集的下限并触发完全重新呈现。

heatmapInstance.configure(configObject)

返回heatmapInstance

在初始化热图实例后重新配置它。触发完全重新渲染。

heatmapInstance.getValueAt(对象)

返回数据点位置处的值

注意：如果点未存储，则返回值是基于渐变混合的插值

heatmapInstance.getData()

返回一个可持久且可重新导入的(使用setData)JSON对象

heatmapInstance.getDataURL()

返回dataURL string

返回的值是热图实例的base64编码dataURL。

heatmapInstance.repaint()

返回heatmapInstance

重绘整个热图画布
