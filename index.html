obj模型获取不到图片

发表于 2019-11-14 | 更新于 2019-12-25 | 分类于 服务端 | 阅读次数

在开发中，前端开启端口8080，webpack用于热更新加载。后端采用3000端口，用于数据请求。
同时设置了Access-Control-Allow-Origin为*。
但是在加载obj时出现了问题，图片加载不出来
The value of the 'Access-Control-Allow-Origin' header in the response must not be the wildcard '*' when the request's credentials mode is 'include'.
原因：

  1. 后台不能使用通配符 ‘*’，否则前端报错；
  2. 请求的origin和后台设置的origin不一致。
